# 한계 (THESIS_LIMITATIONS)

## 데이터

1. **복지 서비스·기관**  
   - `ODCLOUD_SERVICE_KEY` 없으면 8건 시드만 제공.  
   - 공공 API 연동 시에도 장애유형·지역 등은 제목/요약 **키워드 추정**이 포함됨.  
   - 실제 급여·자격 판정은 복지로·소관 기관 기준이 우선.

2. **편의시설**  
   - 별도 공공 API(XML)이며, 복지 서비스 DB와 **통합 ID·연계 없음**.  
   - 개발계정 트래픽 제한(일 100건 등)에 유의.

3. **프론트 폴백**  
   - 백엔드 불가 시 `mock-data.ts` 예시 데이터 — 「예시」 배너로 구분.

## 기능

4. **인증**: 데모 계정만 지원, OAuth·회원 DB 없음.  
5. **개인화**: 즐겨찾기·맞춤 추천·알림 없음.  
6. **영속성**: 백엔드 재시작 시 인메모리 데이터만 유지(시드/재동기화).

## 검증

7. 실제 사용자 테스트·WCAG 공식 감사 전 단계.  
8. 접근성은 가이드라인·`prefers-reduced-motion` 수준의 자가 점검.

## 향후

- PostgreSQL + 배치 동기화(복지로·지자체)  
- taxonomy 공식 코드 매핑 테이블  
- OAuth, 즐겨찾기, 관리자  
- 지도·좌표 기반 주변 시설  
- axe/playwright 접근성 CI
