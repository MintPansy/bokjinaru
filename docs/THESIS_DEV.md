# 개발·실험 메모 (THESIS_DEV)

## 목표

조건 기반 복지 서비스 탐색 UX와 접근성 중심 UI의 타당성 검증.  
장애인 편의시설 공공 데이터 연동을 통해 **시설·서비스 정보를 한 플랫폼에서 탐색**하는 흐름을 검증한다.

## 현재 구현 범위 (MVP)

### 프론트엔드
- 복지나루 UI: 홈, 서비스 검색/상세, 기관, 접근성 안내
- 장애인 편의시설: 검색 폼, 목록, `wfcltId` 기구표 상세 (서버 프록시)
- API 폴백: 백엔드 장애 시 `mock-data` + 안내 배너
- 데모 로그인·마이페이지·문서 초안 플레이스홀더

### 백엔드
- Spring Boot `bokjinaru`: REST API, 인메모리 저장소
- 공공데이터(odcloud) 복지서비스정보 연동 또는 `seed.json` 폴백
- CORS·헬스·메타·필터 API

### 미구현·스텁
- OAuth (`/api/auth/callback` → 501)
- PostgreSQL, 관리자, 즐겨찾기
- 지도·푸시·알림

## 추후 실험 항목

- 필터 정확도·응답 시간 (공공 API vs 캐시)
- 스크린리더·키보드 태스크 완료율 (`/search`, `/disabled-facilities`)
- 모바일 1손 조작·마스터-디테일 레이아웃
- API 장애 시 폴백 UX 인지도·신뢰도
